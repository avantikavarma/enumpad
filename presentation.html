IoT Market
Ticker

A Wi-Fi-enabled Live Market Ticker and Portfolio Desk Toy built on the TI CC3200 — real-time stock prices, threshold alerts, and hardware indicators.

Jia Mohan & Avantika Varma

1. Description

The proposed product is a Wi-Fi-enabled Live Market Ticker and Portfolio Desk Toy. Utilizing the Texas Instruments CC3200, this embedded device acts as a standalone financial monitor. It connects directly to the internet via local Wi-Fi to query real-time stock or cryptocurrency valuations from standard open-source financial REST API endpoints such as Finnhub or CoinGecko.

The core application parses incoming JavaScript Object Notation (JSON) payloads to isolate target market strings, current price indicators, and daily standard evaluation metrics. This compiled state matrix is outputted onto a hardware visual dashboard. Crucially, the system features a hardware-integrated interrupt switch allowing localized user logic routines to cycle real-time data monitoring modules smoothly. If asset limits cross safety barriers, on-board systems activate visual alert paths and buzzer alarms.

2. Design

Functional Specification

The system operates in a continuous polling loop, balanced by hardware delay timers to respect target API endpoints rate limitations. Upon device initialization, system initialization routines handle internal structures, configure interface profiles, check local wireless networks, and initiate an encrypted network pipeline. The core software handles remote endpoint queries, handles JSON validation checks, assesses extracted data parameters against local values, outputs data matrices to display boards, and changes alert lines before waiting for the next cycle.

System Architecture

Architectural Block Diagram Description:

  • Microcontroller Core: TI CC3200 LaunchPad manages networking, API requests, data parsing, and peripheral control.
  • Communication / Cloud Service:
    • Network Processor: CC3200's onboard Wi-Fi connects to the local router.
    • Cloud API: Third-party financial REST API providing data in JSON format over HTTPS.
  • Input Components:
    • Push Button: Connected to a GPIO pin configured as a hardware interrupt (with a pull-up resistor and software debouncing) to cycle through different stock tickers.
  • Output Components:
    • OLED (provided by lab): Connected via the I2C (or SPI) bus to display the ticker symbol and price.
    • RGB LED: Connected to three PWM-capable GPIO pins to visually indicate buy/sell/neutral states.
    • Active Buzzer: Connected to a standard GPIO pin to provide an auditory alert when a threshold is crossed.

3. Implementation Goals

Minimal Goals

  • Establish complete operational initialization parameters between the TI CC3200 board and standard network router points.
  • Formulate and test HTTP GET requests to open network ports, pulling raw strings safely down to structural system buffers and outputting arrays to local UART debugging windows.

Target Goals (Expected Prototype)

  • Incorporate full programmatic string decoding algorithms (e.g., via standard cJSON libraries) inside embedded C modules to extract accurate price variables.
  • Output clean alphanumeric symbols and formatted financial data arrays to standard local LCD displays.
  • Establish system trigger loops matching real-time numbers against hardcoded variables to change state pins on independent RGB modules.

Stretch Goals

  • Configure complex cryptographic certificates and secure SSL/TLS socket connections directly through embedded memory configurations to handle strict production API endpoints.
  • Formulate background asynchronous ISR loops managing the hardware interrupt button to rotate target asset arrays safely without system interruptions.
