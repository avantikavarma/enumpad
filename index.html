IoT Market
Ticker

A Wi-Fi-enabled Live Market Ticker and Portfolio Desk Toy built on the TI CC3200 — real-time stock prices, edge alerts, and integrated sensor tracking controls.

Jia Mohan & Avantika Varma

1. Description

The proposed product is a Wi-Fi-enabled Live Market Ticker and Portfolio Desk Toy. Utilizing the Texas Instruments CC3200, this embedded device acts as a standalone financial monitor. It connects directly to the internet via local Wi-Fi to query real-time stock or cryptocurrency valuations from standard open-source financial REST API endpoints such as Finnhub or CoinGecko.

The core application parses incoming JavaScript Object Notation (JSON) payloads to isolate target market strings, current price indicators, and daily standard evaluation metrics. This compiled state matrix is outputted directly onto an on-board hardware visual dashboard via an OLED display module. Crucially, the system utilizes two unique types of physical sensor and hardware control configurations: an on-board tactile switch (SW[2]) allowing localized user logic routines to signal the UART interface to append additional stock tickers dynamically from a terminal, and an integrated on-board accelerometer allowing gestures or tilts to swipe smoothly through active stock/asset pages. All performance updates, warnings, or rate metrics are displayed exclusively on the localized OLED hardware window.

2. Design

Functional Specification

The system operates in a continuous polling loop, balanced by hardware delay timers to respect target API endpoints rate limitations. Upon device initialization, system initialization routines handle internal structures, configure interface profiles, check local wireless networks, and initiate an encrypted network pipeline. The core software handles remote endpoint queries, handles JSON validation checks, assesses extracted data parameters against local values, outputs data matrices directly onto the hardware OLED display board, and handles page transitions driven by user gesture parameters before starting the next update routine.

System Architecture

Architectural Block Diagram Description:

  • Microcontroller Core: TI CC3200 LaunchPad manages networking, API requests, data parsing, terminal communications, and peripheral control.
  • Communication / Cloud Service:
    • Network Processor: CC3200's onboard Wi-Fi connects to the local router.
    • Cloud API: Third-party financial REST API providing data in JSON format over HTTPS.
  • Input Components (Sensors & Hardware Switches):
    • Tactile Switch (SW[2]): Configured as a hardware interrupt pin to trigger terminal polling listeners, allowing users to enter and load new stock assets dynamically into memory.
    • On-Board Accelerometer: Tracks physical spatial tilt and acceleration changes over I2C/SPI to detect swipe movements, cycling cleanly between stock pages.
  • Output Component:
    • OLED Screen: Connected via the shared serial peripheral bus line to serve as the unified display dashboard for active stock codes, dynamic evaluation prices, threshold metrics, and system status messages.

3. Implementation Goals

Minimal Goals

  • Establish complete operational initialization parameters between the TI CC3200 board and standard network router points.
  • Formulate and test HTTP GET requests to open network ports, pulling raw strings safely down to structural system buffers and verifying response sizes.

Target Goals (Expected Prototype)

  • Incorporate full programmatic string decoding algorithms inside embedded C modules to extract accurate price variables from custom JSON packages.
  • Output clean alphanumeric symbols and formatted financial data arrays exclusively onto the hardware OLED display board.
  • Utilize terminal logging commands triggered via SW[2] interrupts to process fresh user string additions dynamically.

Stretch Goals

  • Configure complex cryptographic certificates and secure SSL/TLS socket connections directly through embedded memory configurations to handle strict production API endpoints.
  • Formulate background asynchronous configuration loops tracking the on-board accelerometer coordinates to allow physical tilt swipes to control visual display windows and show alert flags dynamically.
