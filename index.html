Project Objectives

Quick Description

We are designing a Wi-Fi-enabled IoT desk toy that displays real-time stock prices and trading signals. It uses the TI CC3200 microcontroller to connect to AWS IoT, which triggers a Lambda function to pull live prices from Yahoo Finance. The current price, daily change, and a BUY/SELL/HOLD signal are all rendered on an OLED display — no phone or computer required.

Similar Products & Differences

Similar existing products include the Bloomberg Terminal, the Robinhood mobile app, and DIY LED ticker displays. Ours is different because it is fully self-contained on a $5 embedded device with no app or browser needed, uses a physical accelerometer for gesture-based navigation between stocks, and routes data through a real cloud pipeline (AWS IoT Shadow + Lambda) rather than a direct API call.

Modules and Relationships

  • Cloud Layer: AWS IoT Shadow, Lambda function, Yahoo Finance API. Lambda is triggered when the device POSTs a desired ticker; it fetches the price and writes it back to the shadow's reported state.
  • CC3200 Core: Three internal modules: Wi-Fi Stack (connects and handles TLS), JSON Parser (extracts price/change/alert_color from shadow response), Display Renderer (formats and draws to OLED).
  • Inputs: SW[2] button (hardware interrupt → UART terminal reads new ticker string); On-board accelerometer (X/Y tilt thresholds → page swipe between 5 stocks).
  • Output: 128×128 SPI OLED displaying ticker symbol, price, daily change, BUY/SELL/HOLD signal, and page counter.
  • Data Flow: Device POSTs desired.ticker → Lambda fetches price → shadow reported state updated → device GETs shadow → parses JSON → renders OLED.

Firmware Logical Operations (Flowchart Analysis)

The system runs an infinite polling loop tightly integrated with an interrupt service routine architecture to maintain low-power idle intervals while respecting API limits. The firmware operations are divided into five sequential computational execution steps:

  • System Initialization (INIT): Upon power delivery or hard reset events, the system stabilizes core bus registers—configuring low-level peripherals across GPIO, I2C, SPI, and RGB PWM drivers. Simultaneously, the embedded network stack opens a WPA2 local Wi-Fi handshake protocol and creates a secure SSL/TLS socket instance dedicated to cloud communications.
  • Network Data Acquisition (FETCH_DATA): The CC3200 dynamically assembles an outbound HTTP GET request string mapped specifically to the financial API infrastructure. The core blocks its execution path to wait for incoming response data packets. If a connection failure happens, an error handling subroutine updates the OLED UI with a disconnection diagnostic warning, increments a local wait variable, and loops back to force a fresh network reconnect strategy.
  • Data Stream Processing (PARSE_JSON): Once a response packet is acknowledged, the string buffer passes into a targeted JSON decoding loop. It securely extracts key elements: the alphanumeric stock ticker, current market evaluation price, daily variance ratio, and an epoch time validation string. If structural parsing failures happen, the module intercepts the break state and passes it to the evaluation block using default error flags.
  • Conditional Algorithmic Evaluation (EVALUATE): The extracted pricing engine matches values against user-configured pricing thresholds. If the current price scales equal to or beyond the designated Sell Threshold, the system triggers a programmatic RED Alert latch. If it scales below or equal to the designated Buy Threshold, a GREEN Alert state maps to memory registers. Failing both criteria maintains a baseline neutral buffer state.
  • Hardware Driver Synthesis & Idle States (UPDATE / IDLE): The output pipeline writes live formatted matrices straight down to the physical 128×128 SPI display. Concurrently, it sets the status colors of the RGB peripheral (Red/Green/Neutral) and activates a chime buzzer circuit if anomalous alerts are triggered. Following driver updates, the system slips into a 30-second low-stress sleep delay to comply with standard cloud traffic rate limitations.
  • Asynchronous Interrupt Control (Button Press): Any physical interaction with the hardware button interface fires an immediate hardware edge interrupt handler. This event breaks the standard 30-second polling delay loop, flushes active JSON state registers, and forces an immediate branch back to the FETCH_DATA routine to handle immediate page switching metrics instantly.
